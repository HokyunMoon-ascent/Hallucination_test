# CEP 파인더 할루시네이션 검수 — 실험 계획 요약

## 목적
CEP 파인더가 만든 CEP 카드가 얼마나 할루시네이션을 일으키는지 라이브 데이터로 재고,
어느 정도 품질에 이르면 개선 반복을 멈춰도 되는지를 통계로 가려낸다.

## 판정 지표 (3분리)
| 지표 | 정의 | 목표 | 성격 |
|------|------|------|------|
| Pass rate | pass / 전체 | **≥ 90%** | 생성 품질 게이트 |
| Fail rate | fail / 전체 | **≤ 3%** | 치명 결함(할루시네이션) |
| Suspect rate | 의심 / 전체 | 추적만 | 회색지대 |

> UI 라벨 매핑: 검수 통과=pass / 의심=suspect / 검수 통과 실패=fail

## 2단계 게이트 (실험 절차)

### 1차 — 검수기 신뢰성 검증 (선행 필수)
- 도구: `cep-verify-prompt.md`를 크롬 익스텐션 Claude로 실행
- 측정: **False Pass**(통과라 했으나 실제 할루시네이션), **False Fail**
- 통과 조건: 표본에서 **False Pass ≈ 0**, False Fail 낮음
- 이유: 검수기 판정이 틀리면 통과율 자체가 무의미

### 2차 — 생성 품질 게이트
- 신뢰된 검수기로 pass/fail 카운트 → 통과율 산출
- 중단 조건(아래 "중단 규칙") 충족 시 개선 반복 종료

## 표본 설계
- 단위: CEP **카드** (프로젝트 1개 ≈ 8~16장)
- **서로 다른 프로젝트에서 수집** (한 프로젝트만 보면 운에 좌우됨)
- 최소 표본:
  - **~50장** (약 5개 프로젝트) → 빠른 감 (오차 ±8%p)
  - **~200장** (약 15~20개 프로젝트) → 신뢰 판정 (오차 ±4%p)
- fail율 주장용(Rule of Three, fail 0건 기준):
  - < 5% 주장 → 60장 / < 2% 주장 → 150장

## 중단 규칙 (언제까지 반복?)
> **95% 신뢰구간(Wilson) 하한/상한 기준으로 판정:**
> `Pass CI 하한 ≥ 90%` **AND** `Fail CI 상한 ≤ 3%` 를 동시에 만족하면 중단.

- 관측치가 아니라 CI 경계로 판정 → 소표본 "운 좋은 90%" 조기종료 방지
- 조기 판정이 필요하면 SPRT(순차확률비검정)로 최소 표본에서 도달/미달 결론

## 산출물 흐름
```
cep-verify-prompt.md (크롬 익스텐션 Claude)
   → QA 마크다운 (프로젝트별)
   → 리포트 폴더 Claude Code가 cep-qa-report-template.md 기입
   → 통과율/False Pass 집계 → 중단 규칙 판정 → 백엔드 전달 리포트
```

## 근거 요약
- **90% 목표**: LLM faithfulness(RAGAS·FActScore) 강력 시스템 수렴 구간 85~95%, 95%↑는 수확 체감 → 100% 추구는 낭비
- **fail ≤ 3%**: 할루시네이션=major defect, 제조 AQL 2.5~4% 관점
- **200장**: 관측 90%에서 "진짜 통과율 ≥ 85%"를 95% 신뢰로 주장 가능한 표본 규모

## 초기 설정값 (조정 가능)
- 목표: Pass ≥ 90% / Fail ≤ 3%
- 신뢰수준: 95%
- 최소 표본: 50장(예비) → 200장(확정)

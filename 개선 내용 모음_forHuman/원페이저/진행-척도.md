# 진행 척도 — 원페이저 기준 지금 어디까지 왔나

이 폴더의 시작점은 [CEP 파인더 할루시네이션 개선 One Pager](<CEP 파인더 할루시네이션 개선 One Pager.md>)다.
아래는 그 원페이저를 잣대로, 지금까지의 작업이 어디에 닿아 있는지 보는 척도다.

## 북극성 — 원페이저가 정한 목표

- 큰 방향: Obj3 데이터 차별화 — CEP 카드가 지어낸 이야기 없이 소스 근거에 붙게 만들어 신뢰성을 높인다.
- 할 일 ① CEP 파인더 Phase 내 프롬프트 개선 (비강제)
- 할 일 ② 대시보드 전체 정합률 **70% 이상**

## 출발점 — 원페이저가 찍은 기준선 (21개 프로젝트·210장 스냅샷)

| 항목 | 값 |
|---|---|
| 실패 | 11건 (5.2%) |
| 의심 | 83건 (39.5%) |
| ├ 출처 근거 partial | 49건 (의심의 59%) |
| ├ 검증은 통과했으나 생성 플래그로 강등 | 30건 (36%) — 플래그 기준만 손봐도 크게 줄 여지 |
| └ 카테고리 문제 | 4건 |

이 스냅샷 기준 정합(통과)은 116장 = **약 55%**. 목표는 70%.
근거: [원페이저](<CEP 파인더 할루시네이션 개선 One Pager.md>) · [대시보드](https://hallucination-test.vercel.app/dashboard)

## 지금까지의 진행 — 핵심 척도는 정합(pass)율, 목표 70%

| 시점 | 표본 | pass율 | 메모 | 근거 |
|---|---|---|---|---|
| 7/8 | 20P·200장 | 통과 112 = 56% | 초기 진단, 반복 이슈 4종 정리 | [이슈 리포트](../01_초기-진단/이슈-리포트_0708.md) |
| 7/16 | 14P·140장 | pass 102 = 72.9% | 병목은 카테고리가 아니라 출처 근거 정밀도로 확인 | [7/16 회차](../cep-improve-260716/report.md) |
| 7/20 | 52P·520장 | pass 352 = 67.7% | 1차 합격선 70%까지 pass 12장 부족 | [7/20 회차](../cep-improve-260720/report.md) |
| 7/21 | — | (재측정 없음) | CEP 이론 정합·출력 스키마 개선 (Phase 4 v1.5.0) | [7/21 회차](../cep-improve-260721/report.md) |

회차마다 카드 번들이 달라 pass율을 1:1로 비교하긴 어렵다. 절대값보다 "70%에 얼마나 근접했나"라는 방향으로 읽는다.

## 어떻게 개선하나 — 원페이저 How

- **`cep-prompt-improver` 스킬**: CEP 파인더 json 전달 → 할루시네이션 검수 → 검수 결과를 근거로 Phase 프롬프트를 자가 개선. → [스킬 개발 노트](../03_스킬-개발-노트/cep-prompt-improver-개발노트.md)
- **손보는 지점(원페이저가 지목)**: Phase 3(인용 URL에서 못 찾아도 강제 생성 / 의역 발생), Phase 4(앞서 만든 7W 요소가 강제로 쓰임). → [프롬프트 버전 인덱스](../prompt-version-matrix.md)
- **근거·방향**: [그라운딩 개선 리포트](../02_그라운딩-개선/그라운딩-개선-리포트_0713.md)

## 한 줄 결론

정합률은 원페이저가 찍은 기준선 약 55%에서 최근 회차 67.7%까지 올라와 1차 목표 70% 문턱에 근접했다.
남은 일은 pass를 조금 더 끌어올리는 것과, "검증은 통과했는데 생성 플래그로 강등된 의심"(30건류)의 플래그 기준을 다시 보는 것이다.

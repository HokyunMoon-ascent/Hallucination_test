## Why — 무엇이 문제인가

### 1. 배경 — CEP 파인더에서 할루시네이션이 일어났다

카드를 원본 자료와 대조해 보니, 카드가 **자료에 없는 디테일을 스스로 지어 넣는 일**이 확인됐습니다. 210장 가운데 94장(44.8%)이 검사를 통과하지 못했고(warning 83·fail 11), 그중 90장에 이 계열의 플래그가 붙었습니다.

### 2. 확인 — 시간 · 장소 · 인용 · 행동, 네 방향으로 크게 발생한다

카드를 뜯어 보면 지어내는 방식이 네 갈래로 나뉩니다. 아래 사례는 모두 직접 확인할 수 있습니다. 원본 문장은 URL에서 찾기(Ctrl+F)로 찾아지지만 지어낸 표현은 0건이 나옵니다.

## What — 무엇을 바꾸나

Why에서 확인한 네 유형을 각각 **생성 시점에 차단**합니다. 웹 조사(Phase 3) 바로 뒤에 '증거 분해' 단계(Phase 3.5)를 새로 두고, 카드 생성(Phase 4)을 "자유 작문"에서 "인용 조립"으로 바꾸는 방식입니다.

### 환각은 어느 프롬프트 라인에서 생기나 (As-Is 특정)

As-Is Phase 4(`v.1.0.0_ CEP Trigger Extraction …_KR_0329.md`)에는 **구체어를 소스에 묶어 두는 제약이 한 줄도 없습니다.** 환각을 실제로 부르는 라인은 셋인데, 아래 인용은 KR 프롬프트 원문 그대로입니다.

| 위치 (KR 프롬프트) | 인용 (verbatim) | 왜 환각을 부르나 |
|------|------|------|
| `# 역할` | "소비자가 … 실제 삶의 상황, 트리거, 맥락을 **밝혀내는** 데 있습니다" | 모델을 '발굴자'로 규정 — 소스에 없는 맥락도 '밝혀도' 된다고 읽힘 |
| `# 자연스럽고 현실적인 상황 우선하기` | "보통 사람들이 일상생활에서 실제로 떠올릴 법한 상황을 **작성하세요**" | 소스 근거와 무관하게 '그럴듯한' 장면을 작문하라는 직접 지시 — 시간·장소·발화·행동 창작(①~④)의 원인 |
| `# 형식` | "이 CEP 상황을 직접 뒷받침하는 제품 리서치 결과의 **section_number 값들**" | 근거를 섹션 '번호'로만 요구 — 어떤 문장이 근거인지 소실, 검증 불가(공통·근거 소실) |

To-Be(v1.1.0)는 이 셋을 이렇게 막습니다. (1) 역할을 "지어내지 않고 증거 유닛으로 **조립**하는" 분석가로 다시 규정하고, (2) 자유 작문 지시를 "증거 유닛 선택·결합"으로 바꾸면서 `# Grounding Constraint (CRITICAL)`을 새로 넣고, (3) `section_refs`(번호)를 `rtb`(verbatim 인용문)+`source_ref`+`evidence_unit_ids`로 교체합니다.

### 유형별 차단 매핑

| Why에서 확인한 유형 | 차단 메커니즘 (To-Be) |
|------|------|
| ① 시간 단정 ('첫 주') | 카드의 시간 표현(w7.when)은 **증거 유닛의 인용문에 있을 때만** 기입 — 없으면 비워 둠 |
| ② 없는 장소 ('대기 좌석·테마형 장소') | 장소(w7.where)도 동일 — 인용문에 없는 장소·시설 생성을 프롬프트에서 명시적 금지 |
| ③ 가짜 인용 ('손목도…') | 따옴표 문구는 **RTB 필드로 강제** — 원문 verbatim 복사만 허용, 의역을 따옴표로 감싸는 것 금지 |
| ④ 없는 행동 ('켜둘지 고민') | 카드의 행동 서술은 선택한 인용문의 표현에 대응돼야 함 — 대응 없는 행동 서사 금지 |
| (공통) 근거 소실 | 카드에 `rtb`(인용문) + `source_ref`(섹션) + **URL**이 실려 흘러감 — 이 리포트의 사례표처럼 누구나 찾기로 검증 가능해짐 |

### 파이프라인 변경 (요약)

- **Phase 3.5 신설 (증거 분해)**: 웹 조사 산문에서 verbatim 인용을 유닛으로 구조화합니다 — 인용문 + URL + 7W 부분필드 + 나노인텐트 후보 + KBF 힌트. 인용이 뒷받침하지 않는 필드는 비워 둡니다.
- **Phase 4 (카드 생성)**: 같은 섹션의 유닛 1~3개를 골라 엮어 장면을 조립합니다. 유닛 밖 표현은 만들지 못합니다.
- **Phase 5 (KBF)**: 유닛의 KBF 힌트를 기초 조사와 대조해 확정 (파일 개정은 후속).
- **바꾸지 않는 것**: Phase 3 프롬프트(산문 조사 유지), 이후 단계, 기존 v1.0.0 파일(새 버전 파일로만 개정 — 롤백 가능).

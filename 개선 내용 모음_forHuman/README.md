# 개선 내용 모음 (사람이 읽는 판)

CEP 파인더 할루시네이션을 잡아 온 과정을, 사람이 편하게 읽도록 시점·주제별로 모아 둔 폴더다.
프로젝트 곳곳에 흩어져 있던 진단 노트와 회차 리포트를 여기 한곳에 담고, 문장은 모두 사람 글로 다듬었다.
날것의 실행 데이터(입력 JSON·재제안 결과·로그 등)는 읽기를 방해하니 이 폴더 밖 `_rounds-data/`로 뺐다.

앞으로 새 개선 내용이나 굵직한 수정 사항이 생기면, 날것 데이터는 두고 **사람이 읽을 문서만** 이 폴더에 넣는다.

---

## 흐름 한눈에 보기

처음에는 카드 200장을 훑어 반복되는 문제부터 추렸다(초기 진단).
문제의 뿌리가 "카테고리를 벗어나서"가 아니라 "본문 근거에 제대로 붙지 못해서"라는 게 드러나면서,
근거를 생성 시점에 붙들어 두는 **증거 분해(Grounding)** 쪽으로 방향이 모였다(그라운딩 개선).
이후로는 회차를 돌리며 정합률을 끌어올렸고(7/16·7/20·7/21 회차),
그 과정에서 얻은 프롬프트 개선 요령을 `cep-prompt-improver` 스킬로 정리했다(스킬 개발 노트).

```
초기 진단 (7/7–7/8)
   └─ 그라운딩 개선 (7/10–7/13)
        └─ 회차 반복 검수 (7/16 → 7/20 → 7/21)
             └─ 스킬로 정리 (cep-prompt-improver)
```

## 폴더 규칙

- **사람이 읽을 문서만** 이 폴더에 둔다. 진단·이슈·개선 리포트, 회차 report, 방법론·스킬 노트, 인덱스가 여기 속한다.
- **날것의 실행 데이터**(입력 JSON, 재제안 결과, 집계 원본, 로그 등)는 프로젝트 루트 `_rounds-data/<회차>/`에 둔다.
- **운영·도구 문서**(`README.md`, `cep-verify-prompt.md`, `cep-qa-report-template.md`, `automation/README.md`)는 개선 기록이 아니므로 원래 자리에 둔다.
- 새 문서는 시점·주제에 맞는 하위 폴더에 넣고, 파일을 넣을 때 문장은 사람 글로 다듬는다.

## 4. 우선순위 제안 (백엔드)

1. **검수 커버리지 복구**: 과거/다국어 프로젝트 190카드 미검사 → 재검사 배치 또는 미검사 카드의 사용자 노출 정책 확정.
2. **검수기 신뢰성 검증**(선행): 파일럿에서 False Fail(1289 Card8)·동일소스 상반판정 발견. 52%가 검수기 오류로 **과소평가**됐을 수 있음 → 표본 정밀 QA로 False Fail/False Pass율 측정 후 통과율 재해석.
3. **생성 품질**: warning의 주원인이 "KBF에 원문 근거 없는 제품기능 창작"(파일럿 C2). KBF 근거 필수화가 통과율 개선의 핵심 레버.

## 5. 남은 검증 (권장 다음 단계)

이 집계는 **verdict 값 자체**를 믿고 센 것이다. verdict가 **옳은지**(False Pass/False Fail)는 별도 LLM QA가 필요.
- 검사된 18개 중 대표 표본(~20 카드)을 `cep-verify-prompt.md`로 정밀 QA → False Pass/Fail율 산출 → 위 52%를 보정.

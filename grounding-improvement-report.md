# CEP 파인더 개선 방향 리포트 — 증거 분해(Grounding) 단계 도입

- 대상: CEP 파인더 파이프라인 (CEP_Phase 프롬프트 세트, Phase 3~5 구간)
- 근거 데이터: 완료 프로젝트 21개 / CEP 카드 210장 (`cep_bundle_21proj_210cards.json`) + 재제안 실험 (`repropose/1328.json`)
- 작성일: 2026-07-10
- 관련 산출물: [As-Is/To-Be 비교 맵](https://claude.ai/code/artifact/f5c851d6-5a1c-450a-8a5f-54e45d5540ed) · Phase 3.5/Phase 4 프롬프트 초안 (경로는 How 참조)

> **이 리포트가 다루는 문제** — AI가 만든 CEP 카드가 원본 자료에 없는 디테일("첫 주", "도서관" 같은)을 슬쩍 지어내는 문제입니다.
> `issue-report.md`의 이슈 3("없는 디테일 지어냄")이 바로 이것인데, 이번에 **왜 생기는지(구조적 원인)** 를 찾았고,
> **생기고 나서 잡는 게 아니라 애초에 못 생기게 만드는** 개선안을 제안합니다.

## 세 줄 요약

- 카드 210장 중 **94장(44.8%)이 비통과**(warning 83 + fail 11)이고, 그중 90장에 "출처에 없는 내용" 계열 플래그가 붙어 있습니다. 원인은 특정 제품이 아니라 **파이프라인 구조**입니다 — 21개 프로젝트 전부에서 카드 2~6장씩 나옵니다.
- 웹 조사(Phase 3) 직후에 **'증거 분해' 단계(Phase 3.5)** 를 신설해, 원문 인용(verbatim)을 구조화한 뒤 카드 생성(Phase 4)이 **그 인용 밖으로 못 나가게** 바꿉니다. 사후 교정을 사전 예방으로 옮기는 것입니다.
- 프롬프트 초안 2개와 비교 맵은 이미 만들어져 있습니다. 다음 순서는 **1328 실데이터 dry-run → 파이프라인 코드 연결**입니다.

## Why — 왜 바꾸나

### 1. 규모: 카드 절반 가까이가 "근거 검사"에서 걸린다

210장 전수 검사 결과입니다 (검사 모델: gpt-5.4-nano).

| 판정 | 카드 수 | 비율 |
|------|------|------|
| 통과 (pass) | 116 | 55.2% |
| 의심 (warning) | 83 | 39.5% |
| 실패 (fail) | 11 | 5.2% |

- 비통과 94장 중 **90장에 환각 계열 플래그**(출처에 없는 내용·문구 불일치 등)가 붙어 있습니다. 즉 비통과의 사실상 전부가 "근거 문제"입니다.
- 근거 채점만 따로 보면: 온전함(ok) 147장 / 일부만 맞음(partial) 54장 / 근거 없음(fail) 9장.
- **21개 프로젝트 전부**에서 비통과가 나옵니다(최소 2장, 최대 6장 / 프로젝트당 10장). 특정 제품의 자료 부실이 아니라 만드는 방식의 문제라는 뜻입니다.

참고 — 기존 `issue-report.md`(20proj 기준)는 "지어냄"을 3건으로 셌는데, 그건 검사기가 `fabricated_detail`로 분류한 것만 센 숫자입니다. 같은 성격의 문제("소스에 없는 문구")가 실제로는 `no_source_quote` 30건, `vague_expression` 63건에 흩어져 있어서, **"출처에 없는 내용" 문제의 실제 규모는 이슈 3에서 보이던 것보다 훨씬 큽니다.**

### 2. 패턴: 지어내는 방식이 네 가지로 반복된다

프로젝트 1328(버티컬 마우스, 10장 중 6장 비통과)을 카드 단위로 뜯어 보면, 지어내는 방식이 정해져 있습니다.

| 유형 | 실제 사례 (1328) | 소스에 실제로 있던 것 |
|------|------|------|
| ① 시간 단정 | "재택근무를 시작한 **첫 주**" / "**주말**" | "처음 시작했을 때" (기간 없음) / 시간대 언급 없음 |
| ② 없는 장소 | "일반 사무실/**도서관**" | "회의실/공용 자리, 가족이 자는 집" |
| ③ 가짜 인용 | "'손목도 마우스로 더 안 맞는 것 같다'" (따옴표) | "'마우스까지 인체공학적으로 못 맞추는'" (verbatim) |
| ④ 없는 행동 | "적응 단계에서 **다시 검색**" | "구매/반품 고민"까지만 있음 |

210장 전체의 플래그 문구를 키워드로 훑어보면(근사치, 중복 매칭 포함) 문구 불일치·가짜 인용 계열 ~30건, 시간 단정 계열 ~26건, 없는 장소 ~8건, 없는 행동 ~4건으로, **1328에서 본 패턴이 전체에서도 그대로 반복**됩니다.

### 3. 원인: 산문을 읽고 카드를 "자유 작문"하는 구조

- Phase 3(웹 조사)은 **산문 마크다운**을 내놓습니다. 인용은 "권장" 수준이라 원문 문장이 구조화되어 보존되지 않습니다.
- Phase 4(카드 생성)는 그 산문을 읽고 카드를 **자유롭게 다시 씁니다**. 장면을 그럴듯하게 만드는 과정에서 소스보다 반 발짝 앞선 구체화(첫 주, 도서관…)가 끼어듭니다.
- 카드와 근거의 연결 고리는 **섹션 번호뿐**입니다. 어떤 문장이 근거였는지가 카드에 실리지 않아, 검사 단계에 가서야 문제가 드러납니다.

### 4. 비용: 사후 교정은 사람 손이 든다

지금은 검사기가 걸러낸 카드를 사람이 원문과 대조해 다시 쓰는 방식(repropose)으로 고칩니다. 1328에서는 **10장 중 6장(60%)** 을 손으로 재작성했습니다. 이 비율이 전 프로젝트에서 유지되면(전체 44.8%), 확장할수록 교정 인건비가 카드 생산량에 비례해 늘어납니다.

### 5. 사업 관점: 근거 없는 카드는 재사용이 안 된다

카드에 재인용 가능한 근거(RTB)와 출처(source_ref)가 실려 있지 않으면, 뒤 단계(MA 진단, 콘텐츠 가이드)에서 "이 장면의 신뢰 근거"를 다시 조사해야 합니다. 1328 재제안의 결론이기도 합니다 — **"고해상도 장면마다 신뢰 근거(RTB)를 배치해야 AI 추천 후보군에 들어갈 수 있다."** 근거가 카드에 실려서 흘러가야 이 작업이 공짜가 됩니다.

## What — 무엇을 바꾸나

한 줄: **사후 교정을 사전 예방으로.** 사람이 repropose에서 하던 일(검증된 인용만으로 장면 재조립)을, 웹 조사 직후의 정식 단계 **Phase 3.5(증거 분해)** 로 파이프라인에 내장합니다.

### 변경 4건

| # | 구간 | As-Is | To-Be |
|---|------|------|------|
| 1 | **Phase 3.5 (신설)** | 없음 | 산문에서 verbatim 인용을 뽑아 **증거 유닛**으로 구조화 — 인용문 + 출처 URL + 7W 부분필드 + 나노인텐트 후보 + KBF 힌트 + RTB. 인용이 뒷받침 않는 필드는 **비워 둠** |
| 2 | Phase 4 (카드 생성) | 산문을 읽고 자유 작문 | 같은 섹션의 증거 유닛 1~3개를 **선택·결합**해 장면 조립. 카드의 모든 시간·장소·발화·행동이 인용문에 대응돼야 함. 위 4대 유형은 프롬프트에서 명시적 금지 |
| 3 | Phase 5 (KBF) | 카드만 보고 속성 도출 | 유닛의 KBF 힌트를 기초 조사와 **대조해 확정** (파일 개정은 후속) |
| 4 | 검사기 | 걸러낸 카드를 사람이 사후 교정 | **회귀 감시**로 역할 이동 — fail/warning이 나오면 카드가 아니라 Phase 3.5/4 프롬프트를 고침 |

### 카드에 실리는 정보 (before/after)

```
As-Is  { situation, nanoIntents×3, evidence, section_refs }     ← 근거가 "섹션 번호"로만 남음
To-Be  { situation, w7, nanoIntents×3, kbf_hints,
         rtb, source_ref, evidence_unit_ids }                   ← 인용문이 카드까지 따라감
```

### 사례: CEP 4 (1328) — To-Be가 목표로 하는 출력

- **As-Is 출력 (warning)** — 재택근무를 시작한 ~~첫 주~~, 노트북을 소파/침대에서 쓰다가 목·어깨가 먼저 뻐근해지고 결국 ~~‘손목도 마우스로 더 안 맞는 것 같다’~~고 느껴 교체를 떠올리는 순간
- **To-Be 출력 (§1 인용 2건 조립 = repropose 수정안과 동일)** — **재택근무로 혼자 오래 작업하면서 목·어깨 불편을 먼저 느끼고, 소파·침대 옆에서 노트북을 쓰다 손목이 몸쪽으로 꺾이는 게 반복돼 ‘마우스까지 인체공학적으로 못 맞춘다’고 느껴 교체를 떠올리는 순간**

취소선 = 소스에 없어서 걷어낸 것, 나머지 구체어는 전부 원문 인용에 실재합니다.

### 바꾸지 않는 것

- Phase 3 프롬프트 (웹 조사의 산문 출력은 리서치 품질을 위해 유지 — 구조화는 3.5가 담당)
- Phase 5-1 이후 단계 (단, 카드에 실린 RTB·source_ref를 그대로 재사용할 수 있게 됨)
- 기존 프롬프트 v1.0.0 파일 (개정은 새 버전 파일로만, 롤백 가능)
